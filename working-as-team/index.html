Introducción

Progamación en equipo

La programación en equipo de proyectos hoy en día es fundamental, y los beneficios son múltiples tanto para los desarrolladores como para el proyecto. Si vamos a trabajar en un proyecto en equipo con git deberemos saber:

  • Realizar commits "documentados" al repositorio.

  • Trabajar con ramas.
  • Saber lo que es el 3-way merge.

Trabajando con ramas

Las ramas son bifurcaciones del proyecto, que permiten desarrollar diferentes partes de un software en paralelo. Facilitan el aislar los cambios sin afectar al resto del desarrollo, para posteriormente fusionar la rama con la upstream branch (si así lo deseamos).

Una rama nueva se crea con git branch <nombre>, cambiaremos al contexto de una rama con el comando git checkout <nombre>, y fusionaremos ramas con git merge. Veamos en detalle cada comando.

git branch

Para listar las ramas creadas en el repositorio: 

[~/git-in-practice]$ git branch
* master

La rama principal que siempre tendremos creada se denomina master.

Creamos una rama nueva con el siguiente comando:

[~/git-in-practice]$ git branch testing
[~/git-in-practice]$ git branch
* master
  testing

git branch

Finalmente, borramos una rama con el siguiente comando: 

[~/git-in-practice]$ git branch
* master
  testing
[~/git-in-practice]$ git branch -d testing
[~/git-in-practice]$ git branch
* master

git checkout <branch>

Este comando lo utilizamos para cambiar el contexto y comenzar a trabajar con otra rama: 

[~/git-in-practice]$ git branch
* master
  devel

[~/git-in-practice$ ls
a.txt

[~/git-in-practice$ git checkout devel
Switched to branch 'devel'

[~/git-in-practice$ git branch
  master
* devel

[~/git-in-practice$ ls
a.txt  b.txt

git merge <branch>

Si queremos fusionar una rama concreta con la rama con la que actualmente estamos trabajando, utilizaremos este comando: 

[~/git-in-practice]$ git branch
* master
  devel
[~/git-in-practice$ git merge devel
Updating 211141d..26b8cd6
Fast-forward
 b.txt |    1 +
 1 files changed, 1 insertions(+), 0 deletions(-)
 create mode 100644 b.txt
